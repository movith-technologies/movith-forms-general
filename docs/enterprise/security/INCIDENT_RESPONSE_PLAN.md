# INCIDENT RESPONSE PLAN

## 1. INCIDENT LEVELS

P1:
- Cross-tenant leak
- Security breach
- AI runaway cost

P2:
- Service degradation
- Partial outage

P3:
- Minor bug

---

## 2. RESPONSE FLOW

1. Detect
2. Contain
3. Mitigate
4. Communicate
5. Postmortem

---

## 3. COMMUNICATION

Enterprise clients must:

- Be informed within SLA window

---

## 4. POSTMORTEM

Every P1 must include:

- Root cause
- Fix
- Prevention plan

---

## 5. LOG RETENTION

Incident logs must be preserved.
